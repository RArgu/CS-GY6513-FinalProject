One concern is about the Polygon/Polygonscan API as the main data source. The proposal says it will use the Polygonscan API and Dune Analytics to collect 50–100GB of historical blockchain data. However, the Polygonscan API link provided is not accessible, and free API plans usually have strict rate limits. It is not clearly explained how such a large dataset will actually be downloaded. Without a clear plan for bulk data extraction, this could become a major challenge. It would be helpful to clearly state whether the team plans to limit the dataset to a specific time range or mainly rely on Dune exports. Also, please provide an alternate data source as well, as the link provided for Polygonscan API is not accessible. Adding more details would make the data collection process more realistic and feasible.

Another concern is the number of tools being used. The proposal includes HDFS, MapReduce, Spark SQL, Spark Streaming, Hive, Tez, DASK, MongoDB, Cassandra, Spark MLlib, WEKA, and Matplotlib. Some tools also seem unnecessary. For example, Spark MLlib already supports clustering, so using WEKA separately may not be needed. Spark Streaming might also remove the need for Cassandra. Reducing the number of tools and focusing on core Big Data technologies would make the project more realistic and easier to manage while still showing strong technical skills
